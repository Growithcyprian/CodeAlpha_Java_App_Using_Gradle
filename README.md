\# CodeAlpha Task 3 - Java Application Using Gradle



\## About This Project

This is a Java application built with Gradle as part of my DevOps internship at CodeAlpha. The project demonstrates build automation, dependency management, and CI/CD integration.



\## What This Project Covers

\- Automated Java project builds using Gradle

\- Efficient dependency management through build.gradle

\- CI/CD pipeline using GitHub Actions for continuous delivery

\- Streamlined build and deployment processes

\- Core DevOps principles in Java development



\## Tools Used

\- Java JDK 25

\- Gradle 9.4.1

\- GitHub Actions (CI/CD)

\- Git



\## How to Run This Project



\### Prerequisites

\- Java JDK 17 or higher installed

\- Gradle installed (or use the included Gradle Wrapper)



\### Steps

1\. Clone the repositorygit clone https://github.com/Growithcyprian/CodeAlpha\_Java\_App\_Using\_Gradle.git



2\. Navigate into the projectcd CodeAlpha\_Java\_App\_Using\_Gradle

3\. Build the project

./gradlew build



4\. Run the application

./gradlew run



\## CI/CD Pipeline

This project includes a GitHub Actions workflow that automatically:

\- Compiles the Java source code

\- Runs all tests

\- Packages the application into a JAR file

\- Saves the JAR as a downloadable artifact



The pipeline triggers on every push to the main branch.
